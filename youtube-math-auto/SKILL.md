# YouTube Math Auto — URL만으로 끝내는 진입 스킬

## 정체성

`youtube-math-lesson` 스킬은 자막/요약이 **필수 입력**이라 URL만으로는 작동하지 않는다.
이 wrapper 스킬은 그 갭을 메운다: **URL 1개를 받아 자막을 자동 추출**한 뒤 본 스킬을 호출한다.

박PM 페르소나는 **본 스킬에서 활성화**되며, 이 wrapper는 그 앞단의 "자막 수급 담당"이다.

## 입력

**필수**:
- 유튜브 영상 URL 1개 (`youtube.com/watch?v=...`, `/live/...`, `youtu.be/...`, `/shorts/...` 모두 허용)

**선택**:
- 출력 베이스 디렉토리 (기본: 현재 작업 디렉토리의 `output/`)
- 자막 디렉토리 (기본: `subs/`)
- 학년 힌트 (자동 추정 가능하면 생략 가능)

## 출력 명명 규칙 (반드시 준수)

학습자료 누적 시 일관된 분류·검색이 가능하도록 다음 규칙을 따른다.

**디렉토리·파일명 패턴**:
```
output/{학년}/{단원}/{YYYYMMDD}_{핵심주제}_{video_id8}.html
```

**예시**:
```
output/고1/수학Ⅰ-삼각함수의활용/20260505_사인법칙코사인법칙_W9ReLryy.html
output/고2/수학Ⅱ-도함수의활용/20260512_접선의방정식_AbCd1234.html
output/중3/원의성질/20260520_원주각정리_XyZ987Ab.html
```

**구성 요소**:
- `학년`: `중1`, `중2`, `중3`, `고1`, `고2`, `고3` 중 하나 (자동 정규화). 미상이면 `미분류`.
- `단원`: stage1에서 추출한 한국 교육과정 단원명. `[고1-수학Ⅰ-삼각함수의 활용]` 같은 원형은 자동으로 `수학Ⅰ-삼각함수의활용`으로 정규화 (학년 코드 제거, 공백 제거, 30자 제한).
- `YYYYMMDD`: 생성 일자.
- `핵심주제`: stage1의 핵심개념 1~2개를 16~24자로 압축. 예: "사인법칙코사인법칙", "이차함수의최댓값". `·`, `,`, `()` 등 구두점 제거.
- `video_id8`: YouTube video ID 앞 8자 (충돌 방지용).

**Windows 호환**: `\ / : * ? " < > |` 등 금지 문자는 자동 제거. 한글·로마숫자(Ⅰ Ⅱ Ⅲ)는 그대로 유지.

**자동화**:
- 경로 빌드는 `scripts/output_path.py` 의 `build_output_path()` 사용.
- 학년·단원·핵심주제 결정은 본 스킬(`youtube-math-lesson`) stage1 산출물에서 가져옴 (별도 추측 금지).

## 학습자 수준 가이드 (반드시 본 스킬 호출 시 전달)

**대상 학습자**: **중수준 — 하위 40% 학생까지 수월하게 따라갈 수 있어야 함**.

### 핵심 원칙: "강사가 생략한 부분을 자료가 메운다"

영상 강사는 시간 제약·청중 가정 때문에 다음과 같이 풀이를 자주 압축한다:
- *"이거 그 유명한 공식이죠"*, *"이건 알지?"*, *"당연히 이렇게 되니까"*
- *"여기는 그냥 이 공식 쓰면 돼요"*, *"증명은 생략"*
- 그래프·도형을 손짓으로 설명하고 식 유도는 건너뛰기
- 용어(중근, 다중도, 외접원, 변곡점 등)를 정의 없이 사용

본 스킬의 **개념 설명 섹션(STAGE 3 ✍️ 개념작가)** 은 이런 생략을 모두 메워야 한다. 자막에서 강사가 "당연하다"고 넘긴 모든 공식·정리·용어에 대해 다음 4가지를 빠짐없이 작성한다:

1. **출처/유도**: 이 공식·정리는 어디서 왔는가? 이전 학년·단원의 어떤 정리에서 도출되는가?
2. **구조**: 식의 각 부분이 무엇을 의미하는가? 변수·상수·계수가 그래프의 무엇에 대응하는가?
3. **적용 조건과 한계**: 언제 쓸 수 있고 언제 쓸 수 없는가? 반례는?
4. **시각적 직관**: 그래프·도형으로 떠올릴 수 있는 그림. 비유 1개 이상.

### 구체적 작성 규칙

- **공식이 등장하면**: 결과만 적지 말고, **유도 1번은 반드시** 보여준다 (적분 직접 계산 1번, 작도/분할 그림으로 1번 등). 학생이 "아 이래서 나온 식이구나" 를 1번이라도 경험하게.
- **용어가 첫 등장 시**: 정의 + 예시 + 반례를 **묶어서** 한 번에 제시. 예: "중근이란 ... (예: $(x-2)^2$의 $x=2$). 단근과 다른 점: 그래프가 $x$축을 뚫지 않고 닿고 돌아온다 — 그림 ⟨…⟩".
- **강사의 비유·직관**(예: "특수 풀이", "강물·다리")이 등장하면, 그 비유의 **수학적 근거**를 함께 제시. 비유는 이해 다리고, 정의는 단단한 땅.
- **풀이 단계 비약 금지**: 한 줄에 두 단계가 압축되어 있다면 두 줄로 분리. "$\therefore$" 만 적고 결론으로 점프하는 표현 ❌.
- **선수 학습 한 번 더 짚기**: 영상이 "중3 때 배운 거"로 넘어가도, 본 자료는 그 중3 정리를 1~2문장으로 다시 환기.
- **문단 호흡**: 한 문단 2~3문장. 길어지면 줄바꿈으로 분리. 친근한 어조 ("...할 수 있어요") 유지.

### 자가 점검 체크리스트 (STAGE 3 마무리 시)

- [ ] 자막에 등장한 강사의 "당연" 표현(*"이거 알지?", "그냥 이거 쓰면 돼"*)을 모두 식별했는가?
- [ ] 각 표현이 가리키는 공식·정리에 대해 위 4단계(출처/구조/조건/직관)를 작성했는가?
- [ ] 영상에 "유명한 공식"이 등장했다면, 그 공식의 유도(증명) 1번을 자료에 포함했는가?
- [ ] 새 용어가 등장한 곳마다 정의·예시·반례 묶음이 있는가?
- [ ] 풀이 STEP에서 "왜 이 단계가 가능한지" 가 빠진 곳은 없는가?
- [ ] 중수준 학습자가 정자세로 한 번 읽고 이해 가능한 호흡인가?

### 본 스킬 호출 시 전달 문구 (필수 포함)

`youtube-math-lesson` 스킬을 호출할 때 다음 가이드라인을 입력에 포함한다:

> **학습자 수준**: 중수준(하위 40%) 학생도 따라갈 수 있도록 작성. 영상에서 강사가 "당연하다"고 생략한 모든 공식·정리·용어를 개념 설명에서 풀어쓸 것. 결과 공식만 제시 ❌, 유도 1번은 반드시 포함. 선수 학습은 1~2문장으로 환기. 풀이 STEP은 한 줄에 한 단계로 분리, 비약 금지.

## 카탈로그 자동 갱신 (반드시 실행)

새 학습자료를 생성한 직후 다음을 한 번 실행하면 **두 카탈로그가 동시에 갱신**된다:

```bash
python scripts/regen_index.py output/
```

생성되는 파일 두 개:

1. **`output/INDEX.md`** — 마크다운 카탈로그
   - 학년 → 단원 → 날짜(역순) 그룹핑
   - 각 행: 생성일, 주제, 영상 제목·채널 링크, HTML 파일 링크
   - 텍스트 빠른 확인용 (터미널·에디터에서 바로 읽기)

2. **`output/index.html`** — 시각적 갤러리
   - YouTube 썸네일 카드 그리드 (`https://img.youtube.com/vi/{video_id}/mqdefault.jpg`)
   - 학년·단원 필터 버튼 (학년 선택 시 단원 필터 동적 갱신)
   - 제목·주제·채널 실시간 검색
   - 다크모드 토글 (localStorage 저장)
   - 키보드 단축키: `D` (다크모드), `/` (검색 포커스)
   - 각 카드에서 "📖 학습자료 열기" + "▶ 원본 영상" 두 버튼
   - **중복 영상 자동 감지**: 같은 video_id로 여러 파일이 있으면 상단에 ⚠️ 알림 박스 표시

브라우저에서 `output/index.html` 을 열면 모든 학습자료를 한눈에 둘러볼 수 있다.

## 사전 요구사항 (1회만 설치)

```bash
pip install yt-dlp
```

설치 안 됐는지는 `python -m yt_dlp --version` 으로 확인. 미설치 시 사용자에게 설치 명령을 안내하고 작업 보류.

## 워크플로우 (4단계)

### Step 1 — URL 검증 + 영상 메타데이터 확인

1. **video_id 추출** (정규식): `(?:v=|/live/|youtu\.be/|/embed/|/shorts/)([A-Za-z0-9_-]{11})` 또는 `output_path.extract_video_id_from_url()`.
2. **YouTubeData MCP `get_video_details`** 호출 → `duration`, `liveBroadcastContent`, `title`, `channelTitle`, `defaultLanguage` 확보.
3. **중복 영상 감지** (반드시 실행):
   ```python
   from output_path import find_existing
   matches = find_existing("output", video_id)
   ```
   - 매치가 1개 이상이면 사용자에게 알림: *"이 영상은 이미 처리됨 → {경로}. 어떻게 할까요?"*
     - **건너뛰기**: 작업 중단, 기존 파일 안내
     - **덮어쓰기**: 같은 경로로 재생성 (stage1 재실행)
     - **새 버전**: 파일명에 `_v2`, `_v3` 접미사 추가
   - 매치 없음 → 정상 진행
4. **차단 조건**:
   - `liveBroadcastContent: live` (진행 중) → 즉시 거절. 메시지: *"진행 중인 라이브 스트림은 자막이 생성되지 않아 학습자료 제작이 불가합니다. 라이브 종료 후 VOD에 자동자막이 붙으면(보통 수 시간~수 일) 다시 시도해 주세요."*
   - `duration > 60분` → 사용자 확인 요청. 토큰 비용 안내 후 진행 여부 결정.
5. **수학 영상이 아닐 가능성** → 제목·채널·설명을 보고 명백히 수학이 아니면 사용자 확인.

### Step 2 — 자막 추출 (자동화 핵심)

스크립트 실행:
```bash
python scripts/fetch_subtitle.py "<URL>" --out subs/
```

이 스크립트가 하는 일:
1. yt-dlp 호출 (한국어 우선: `ko, ko-KR, ko-orig`, 폴백: `en, en-US`)
2. 다운로드된 `.vtt` 파일을 평문으로 정제 (슬라이딩 윈도우 중복 제거 + 인라인 타이밍 태그 제거 + 30초 단위 타임스탬프 마커 부착)
3. 결과 `.txt` 경로를 stdout으로 출력

**자막 추출 실패 시** (자막 트랙이 전혀 없는 경우):
- 스크립트 종료 코드 ≠ 0
- 사용자에게 두 옵션 제시:
  - **옵션 A**: 본인이 영상 핵심 3~5줄 요약을 입력 → 그 요약으로 본 스킬 호출
  - **옵션 B**: 작업 중단

### Step 3 — youtube-math-lesson 스킬 호출

다음 정보를 정리해 본 스킬을 호출:
- URL
- 영상 메타데이터 (제목, 채널, 길이, 학년 추정)
- 자막 텍스트 파일 경로 (또는 사용자 요약)
- **출력 파일 경로**: `scripts/output_path.py` 의 `build_output_path()` 로 미리 계산하여 본 스킬에 전달.
  본 스킬의 stage1이 산출하는 학년·단원·핵심개념을 사용해야 하므로, **stage1 결과를 받은 후에 경로를 빌드**한다.
- 추가 가이드 (있으면): 학년, 모드(빠른/정밀), 강조 개념 등

본 스킬이 stage1~5를 자동 진행한다. 박PM 페르소나, 8개 페르소나 워크플로우, 3중 검증 모두 본 스킬에서 처리.

### Step 4 — 카탈로그 갱신 + 결과 인도

1. **INDEX.md + index.html 동시 갱신**:
   ```bash
   python scripts/regen_index.py output/
   ```
   `output/` 전체를 스캔해 마크다운 인덱스와 HTML 갤러리를 재생성. 중복 영상이 있으면 콘솔에 보고됨.

2. **사용자 안내**:
   - 새로 생성된 HTML 파일 경로 (예: `output/고1/수학Ⅰ-삼각함수의활용/20260505_사인법칙코사인법칙_W9ReLryy.html`)
   - 갱신된 카탈로그 두 개:
     - 텍스트: `output/INDEX.md`
     - 갤러리: `output/index.html` (브라우저로 열기 권장)
   - 자막 추출 통계 (cue 수, 정제 후 글자 수)
   - 라이브/긴 영상/자동자막/중복 등 주의사항

## 절대 금지

- ❌ 자막이 없는 영상에 대해 제목·설명만으로 추측 학습자료 생성
- ❌ yt-dlp 실패를 무시하고 본 스킬에 빈 자막 전달
- ❌ 본 스킬(`youtube-math-lesson`)의 페르소나·검증 단계 우회
- ❌ "수능 대비" 외 라벨로 실재 기출문제 출처 위조 (본 스킬과 동일 규칙)

## 참조 파일

```
youtube-math-auto/
├── SKILL.md                  ← 이 파일 (진입점)
└── scripts/
    ├── fetch_subtitle.py     ← yt-dlp 호출 + VTT 정제 통합 스크립트
    ├── vtt_to_text.py        ← VTT → 평문 변환 모듈 (fetch_subtitle.py가 import)
    ├── output_path.py        ← 출력 경로 명명 규칙 (build_output_path 함수)
    └── regen_index.py        ← output/INDEX.md 자동 재생성 스크립트
```

## 관계도

```
사용자 (URL만 던짐)
    │
    ▼
youtube-math-auto (이 스킬)
    │  Step 1~2: 자막 자동 수급
    │  Step 3: 본 스킬 호출
    ▼
youtube-math-lesson (본 스킬, 무수정)
    │  박PM + 8 페르소나 + 5 stage
    ▼
HTML 학습자료 (사용자에게 인도)
```

## 디버깅 팁

- 자막 다운로드만 따로 테스트:
  ```bash
  python scripts/fetch_subtitle.py "https://www.youtube.com/watch?v=W9ReLryydQg" --out /tmp/subs
  ```
  마지막 줄에 출력된 `.txt` 경로를 cat 해서 결과 확인.

- 자막은 있는데 정제가 이상하면 `vtt_to_text.py` 의 `dedupe_sliding` 동작 점검.

- 라이브 종료 직후 VOD는 자동자막 생성에 시간이 걸린다. 30분~수 시간 후 재시도.
